# SQL Ecommerce Sales Analysis

## Project Overview
This project analyzes ecommerce sales data using SQL to identify revenue trends, top-performing categories, customer behavior, and country-wise sales performance. The goal of this project is to derive actionable business insights from raw sales data using SQL.

## Tools Used
- MySQL
- Excel / CSV
- GitHub

## Dataset
The dataset contains:
- orders data
- users data
- products data

## Project Assets
- SQL Queries
- Output Screenshots

## Analysis Performed
- Total Revenue
- Total Orders
- Average Order Value
- Sales by Category
- Sales by Country
- Yearly Sales Trend
- Top Customers

## SQL Concepts Used
- Data aggregation using GROUP BY
- Multi-table joins to combine datasets
- Window functions for running totals and ranking

## KPIs
- Total Revenue: **₹15,95,831**
- Total Orders: **10,000**
- Average Order Value: **₹160**
- Total Quantity Sold: **15,504**

## Key Insights
- Beauty category generated the highest revenue (35% of total sales)
- Customers purchase ~1.5 items per order, indicating low basket size and opportunity for cross-selling
- Electronics category contributed significantly lower revenue compared to other categories, indicating potential pricing or demand issues
- Revenue is concentrated in a few key countries
- Average order value is low (~₹160), indicating customers purchase low-value items per order, suggesting an opportunity for upselling

## Recommendations
- Optimize pricing and introduce targeted promotions for underperforming categories to increase their contribution to total revenue
- Increase average order value using upselling and bundle offers
- Focus marketing on top-performing countries while improving weak regions

## Conclusion
The analysis highlights key opportunities to improve revenue through better pricing strategies, customer targeting, and increasing average order value, enabling more data-driven business decisions.
